# CSR_Seed -- Navigation Guide

You are working with a Corpus Semantic Registry (CSR) seeded for a new theory / framework / vocabulary. The registry is empty; you populate it.

## Framing

CSR is a **registry layer over a corpus of documents.** Documents (the corpus) own the prose, math, and definitions. CSR (the registry) owns the names, versions, dependencies, status, and audit trail. One symbol, one definition home, one registry identity.

The CSR convention is: documents define; CSR identifies. When two documents both touch a concept, the registry resolves which one is canonical. When a symbol moves or is renamed, the registry records the predecessor edge and downstream consumers don't churn.

## Directory layout

```
csr/
  registry/           <-- you edit these .csr files
    documents.csr     <-- one document per block; index of all docs
    symbols_core.csr  <-- general-purpose symbol bucket
    <Namespace>.csr   <-- lift here once a namespace has >5 symbols
    aliases.csr, collisions_resolved.csr, invariants.csr,
    migration.csr, predecessors.csr, vocabulary_disambiguation.csr
  build/              <-- compiled output; regenerated by `python3 tools/csr.py build`
    CSR.lock.json     <-- structured lockfile
    CSR.overview.md   <-- one-page summary
    CSR.symbols.md    <-- all symbols
    CSR.documents.md  <-- all documents
    CSR.collisions.md, CSR.status.md, CSR.dependencies.dot,
    CSR.wiki.md, CSR.registry.html
  queue/              <-- workflow surface (open tasks)
  quarantine/         <-- inbound unclassified items
  tools/              <-- compiler source (csr.py CLI, csr_compile.py, ...)
  bump-policy.yaml    <-- version bump rules
  csr_edit_log.jsonl  <-- audit trail (one JSON line per edit)
  csr_config.yaml     <-- paths and notation config (v0.2; see below)
  csr_schema.yaml     <-- enums and validation policy (v0.2; see below)
```

## How to use

**Add a document.**

Append to `registry/documents.csr`:

```
document MyDoc:
  id: csr.document.MyDoc
  document_class: framework
  namespace: MyDoc
  display_name: "My document"
  version: v0
  status: candidate
  source_path: docs/my_doc_v0.md
  defines:
    - csr.MyDoc.my_symbol
  publication_mode: internal
  last_verified: 2026-06-03
```

`source_path` is relative to the corpus root (the directory holding `csr/`). The compiler walks up from `csr/` to find it.

**Add a symbol.**

In `registry/symbols_core.csr` (or a dedicated `registry/MyDoc.csr` once you have several):

```
symbol my_symbol:
  id: csr.MyDoc.my_symbol
  namespace: MyDoc
  framework_layer: domain
  owning_document: csr.document.MyDoc
  display_name: my symbol
  type: concept
  status: candidate
  definition_home: csr.document.MyDoc@v0#section.1.my_symbol
  definition: "..."
  definition_hash: auto
  source_anchor:
    document: csr.document.MyDoc
    version: v0
    section: "1"
    anchor: my_symbol
    source_hash: auto
    section_hash: auto
  verification:
    state: argued
    method: prose
    last_verified: 2026-06-03
```

**Build.** Run `python3 tools/csr.py build` (Windows) or `python3 csr.pyz build` (any platform). Reports `inputs=N errors=E warnings=W` and writes the build/ artifacts.

**Lookup.** `python3 csr.pyz lookup "phrase"` matches across id, display_name, definition, aliases. Use this BEFORE adding a new symbol to check for collisions.

**Walk dependencies.** `python3 csr.pyz deps csr.MyDoc.my_symbol --depth 2` shows the dependency tree; `--inverse` walks `used_by`.

## Closed enums

These are baked into `tools/csr_compile.py` (constants at the top). For a generic engine refactor (v0.2), they'd move to `csr_schema.yaml`. For now you can edit them directly.

- **Status taxonomy (7):** `draft, candidate, provisional, canonical, deprecated, rejected, deferred`
- **Framework layer (6):** `procedure, algebra, representation, domain, cross_layer, registry`
- **Symbol types (8):** `concept, operator, structure, schema, axiom, theorem, predicate, open_problem`
- **Verification states (5):** `declared, argued, tested, formalized, proved`
- **Verification methods (6):** `prose, empirical, theorem, simulation, Lean, compiler_check`
- **Relation types (9):** `depends_on, refines, instantiates, used_by, conflicts_with, equivalent_to, contains, component_of, xref_only`

## Validation rules (CSR codes)

The compiler emits diagnostic codes when registry state violates discipline. Selected codes:

- `CSR001` duplicate symbol id
- `CSR003` unresolved dependency
- `CSR006` invalid status
- `CSR007` canonical without verification
- `CSR008` dependency cycle (on `depends_on`, `refines`, `instantiates`, `contains`)
- `CSR010` missing source_anchor on an active symbol
- `CSR011` invalid verification.last_verified
- `CSR017` namespace/framework_layer conflation
- `CSR023` document declared with no source_path
- `CSR027` notation collision: symbol claims a plain Latin capital that's conventionally reserved

## Notation conventions (CSR027)

The compiler ships with a default reserved-letter set: `{A, V, G, O, F, K, L}`. Override in `tools/csr_compile.py` (constant `NOTATION_RESERVED_PLAIN_CAPITALS`) for your conventions, or set a symbol's `notation.convention_status` to `convention_adapted` or `aesthetic_override` with a rationale to suppress the warning case-by-case.

## Workflow

1. Inbound thought arrives, write a file in `quarantine/T-YYYY-MM-DD-NNN.md` with: proposed name, kind (symbol/document/note), related_symbols, related_documents, parking_reason.
2. Run the gluing test: does it preserve writer discipline; does it compose with existing apparatus; does it introduce a new structure that doesn't fit existing entries?
3. If admit: move to `queue/T-YYYY-MM-DD-NNN/intake.md`, draft the registry block, append to the appropriate .csr file, run `python3 tools/csr.py build`, write `queue/T-.../closure.md`.
4. If reject: leave a closure file in quarantine explaining why.

## v0.2 roadmap

The seed currently inherits the compiler from a production CSR deployment with three couplings to corpus-specific defaults:

1. **Path assumption:** `tools/csr_extract.py` walks up two levels from `csr/` to find the corpus root. Replace with `csr_config.yaml: corpus_root_levels_up: N`.
2. **Hardcoded enums:** see "Closed enums" above. Move to `csr_schema.yaml`.
3. **Notation reserved set:** hardcoded `{A,V,G,O,F,K,L}`. Move to `csr_schema.yaml: notation.reserved_plain_capitals: [...]`.

The seed ships with `csr_config.yaml` and `csr_schema.yaml` as design specs; the compiler does not yet read them. Wiring them through is the v0.2 work.
