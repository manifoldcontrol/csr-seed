how drift detection works

for corpora where prose and code share a vocabulary that an ai agent also has to navigate.

1register

documents own the prose and the definitions. the registry gives each symbol a definition home and tracks what depends on it.

2pin

every definition home gets a sha256, and the build writes the lockfile that every rendered view reads from. validation runs 20+ typed error codes on each compile.

3detect

any change to a pinned source flags every dependent symbol with a CSR004 drift diagnostic until it is re-verified and re-pinned.
