csr-seed

a corpus semantic registry: documents define, the registry identifies, with hash-pinned drift detection.

two example corpora, compiled and rendered. each is a browsable registry: symbols, definition homes, dependency edges, verification states, and the collisions and aliases the registry resolves.

relay — an api project

an auth design doc and an api spec. two teams used session for different things; the registry records the collision and its resolution. api_key was renamed access_token; the alias keeps old references resolving.

gambit — a card game

a rulebook and its engine. the same machinery on prose-vs-code drift: capture collides between the two, resolved on the rulebook sense.

the point these pages can't show statically: edit a source document and rebuild, and every symbol whose definition home changed is flagged with a CSR004 drift diagnostic until re-verified and re-pinned. that check runs in ci on every commit.

source and the 60-second walkthrough: github.com/manifoldcontrol/csr-seed
